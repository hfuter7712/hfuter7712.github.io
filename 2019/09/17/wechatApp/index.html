微信小程序开发复习(一)

发表于 2019-09-17 | 分类于 coding

App.js
作用： 小程序逻辑撰写
App.json
作用:小程序公共设置
"pages":[]
配置页面路径
每个自定义页面都必须在pages页面被注册，否则访问未注册页面会报错

"window":{}
配置窗口表现
navigationBarBackgroundColor 导航条背景色
navigationBarTitleText 导航条文字
navigationBarTextStyle 导航条文字颜色

"tabBar":{}
配置标签导航 

"tabBar": {
    "selectedColor":   //标签导航选中时文字颜色
    "backgroundColor":   //标签导航背景色  
    "borderStyle":     //标签导航上边框色
    "list":[{
       "pagePath":       //跳转页面路径
       "text":           //标签导航名称
       "iconPath":       //默认时图标
       "selectedIconPath":   //选中后的图标  
       },
       {
       "pagePath":
       "text":
       "iconPath":
       "selectedIconPath":  
       }]
}

"networkTimeout":{}
配置网络超时

"debug":true/false
配置debug模式，是否开启

App.wxss
作用:小程序公共样式表

utils文件夹
存放工具栏的js函数
定义完函数之后需要通过module.exports将定义的函数名称注册出来，才能在其他页面使用

App.js 生命周期函数
onLaunch函数
当小程序初始化完成时，会触发，全局只触发一次

onShow函数
当小程序启动，或者从后台进入前台显示时触发

onHide函数
当小程序从前台进入后台触发该函数

onError函数
用来监听小程序脚本或者API是否发生错误，发生错误时返回错误信息

页面 Page生命周期函数
onLoad函数
一个页面只会调用一次

onShow函数
页面显示，每次打开页面都会调用一次

onReady函数
页面初次渲染完成，一个页面只会调用一次，代表页面已经准备妥当

onHide函数
页面隐藏,当navigateTo或者底部Tab切换时调用

onUnload函数
页面卸载，当redirectTo或者navigateBack时调用

onPullDownRefresh函数
监听用户下拉刷新事件

onShareAppMessage
用户分享，只有定义了此事件处理函数，右上角才会显示“分享”按钮，用户点击“分享”按钮的时候会调用

自定义函数
example: 

Fun: function() {
//函数内容   
}

绑定数据
用双大括号表示
example:

条件渲染
wx:if=""
判断是否需要渲染改代码 

<block wx:if="{{true}}">  
  内容
</block>

区块条件渲染

wx:for="
用wx:for绑定一个数组，就可以使用数组中的各项数据重复渲染该组件 

<view wx:for="{{array}}">
{{index}}: {{item.message}}  
</view>

区块列表渲染 

<block wx:for="{{[1,1,5,4,1,4]}}">
  <view>{{index}}:</view>  
  <view>{{item}}</view>
</block>

使用wx:for-index可以指定数组当前下标的变量名
使用wx:for-item可以指定数组当前元素的变量名
wx:for-index="idx" wx:for-item="itemName"

wx:key指定唯一标识符

import 和 include
import引用模板文件
include将整个文件除了</template>之外的进行引用

view视图容器
是WXML界面布局的基础组件,和HTML里面的div功能类似

scroll-view可滚动视图
允许视图区域内容横向滚动或者纵向滚动

swiper滑块视图容器
用来在指定区域内切换内容的显示
